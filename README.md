# TG WS Proxy

**TG WS Proxy** — локальный MTProto-прокси для Telegram Desktop, который передаёт трафик через WebSocket. Такое решение позволяет обеспечить стабильное подключение к серверам Telegram, сохраняя штатное MTProto-шифрование и исключая необходимость использования сторонних прокси.

# Windows: начало работы

Перейдите на страницу **Releases** и загрузите исполняемый файл, соответствующий вашей операционной системе.

Доступны следующие варианты:

- `TgWsProxy_windows.exe` — Windows 10 и новее (x64);
- `TgWsProxy_windows_arm64.exe` — Windows 10 и новее (ARM64);
- `TgWsProxy_windows_7_64bit.exe` — Windows 7 (x64);
- `TgWsProxy_windows_7_32bit.exe` — Windows 7 (x86).

При первом запуске приложение автоматически откроет руководство по подключению Telegram Desktop. После завершения настройки программа продолжит работу в фоновом режиме и будет доступна через значок в системном трее.

## Меню приложения

Через меню значка в области уведомлений доступны основные функции:

- **Открыть в Telegram** — автоматически создать ссылку `tg://proxy` и передать её клиенту Telegram.
- **Скопировать ссылку** — сохранить ссылку подключения в буфер обмена.
- **Перезапустить прокси** — выполнить перезапуск локального MTProto-прокси без закрытия программы.
- **Настройки...** — открыть окно конфигурации, изменить параметры работы и включить автоматическую проверку обновлений.
- **Открыть логи** — перейти к журналу работы приложения.
- **Выход** — остановить прокси и полностью завершить работу программы.

# Подключение Telegram Desktop

## Автоматическая настройка

Наиболее простой способ подключения — воспользоваться встроенной функцией автоматической настройки.

Для этого:

1. Нажмите правой кнопкой мыши по значку TG WS Proxy в системном трее.
2. Выберите пункт **«Открыть в Telegram»**.
3. Подтвердите использование прокси.

Если Telegram не обработал ссылку автоматически:

1. Откройте меню программы.
2. Выберите **«Скопировать ссылку»**.
3. Отправьте её в чат **«Избранное»**.
4. Перейдите по полученной ссылке.
5. Подтвердите подключение.

## Настройка вручную

Если автоматическое подключение недоступно, необходимые параметры можно задать самостоятельно.

Откройте раздел:

```text
Telegram
→ Настройки
→ Продвинутые настройки
→ Тип подключения
→ Прокси
```

Создайте новое MTProto-подключение и заполните следующие поля:

| Параметр | Значение |
|----------|----------|
| Тип | MTProto |
| Сервер | `127.0.0.1` (или другой адрес из конфигурации) |
| Порт | `1443` (либо пользовательское значение) |
| Secret | секретный ключ из настроек приложения или файла логов |
# Как это работает

```text
Telegram Desktop
        │
        ▼
Локальный MTProto Proxy
        │
        ▼
WebSocket (TLS)
        │
        ▼
Telegram Data Center
```

После запуска TG WS Proxy принимает подключения от Telegram Desktop и самостоятельно организует дальнейшую передачу трафика через WebSocket к нужному серверу Telegram.

Во время работы приложение выполняет следующие действия:

1. Запускает локальный MTProto-прокси на заданном адресе и порту.
2. Принимает и обрабатывает входящие подключения Telegram Desktop.
3. Определяет необходимый дата-центр Telegram (DC ID), анализируя MTProto Obfuscation Init Packet.
4. Устанавливает защищённое WebSocket-соединение (TLS) с соответствующим дата-центром.
5. Если подключение через WebSocket недоступно (например, сервер отвечает кодом **HTTP 302 Redirect**), автоматически переключается на Cloudflare Proxy либо использует прямое TCP-подключение.

---

> [!IMPORTANT]
>
> ### Не загружаются фотографии или видео?
>
> Если после подключения через TG WS Proxy Telegram перестал отображать фотографии, видеозаписи или другой медиаконтент, сначала проверьте настройки списка IP-адресов дата-центров.
>
> Откройте раздел **DC → IP** и оставьте только следующую запись:
>
> ```text
> 4:149.154.167.220
> ```
>
> Если это не помогло, полностью очистите содержимое поля и повторите попытку.
>
> Подобная ситуация иногда возникает на аккаунтах без активной подписки **Telegram Premium**.
>
> Если проблема сохраняется, рекомендуется использовать собственный Cloudflare-домен, выполнив настройку по инструкции:
>
> - [Настройка Cloudflare Proxy](./CfProxy.md)

# Автоматическая сборка

Репозиторий уже содержит все необходимые конфигурационные файлы для автоматической сборки приложения.

Создание исполняемых файлов выполняется с помощью **PyInstaller**, а процесс публикации полностью автоматизирован через **GitHub Actions**.

Используемые спецификации:

- `packaging/windows.spec`
- `packaging/macos.spec`
- `packaging/linux.spec`

Workflow автоматической сборки расположен по пути:

- `.github/workflows/build.yml`

Это позволяет собирать актуальные версии проекта автоматически после внесения изменений.

# Поддерживаемые платформы

На данный момент проект предоставляет готовые сборки для следующих операционных систем:

| Платформа | Версия |
|-----------|---------|
| Windows x64 | Windows 10 и новее |
| Windows ARM64 | Windows 10 и новее |
| Windows x64 | Windows 7 |
| Windows x86 | Windows 7 |
| macOS Intel | 10.15 и выше |
| macOS Apple Silicon | 11.0 и выше |
| Linux x86_64 | AppIndicator обязателен |

> [!NOTE]
>
> Для Linux требуется установленный **AppIndicator**, так как без него значок приложения не будет отображаться в системном трее.
